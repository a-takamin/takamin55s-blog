## CDN とは何か。何を解決するのか
CDN とは Contents Delivery Network の略で、ブログや動画などのコンテンツを配信する際に、ホストする本体サーバーに加えてユーザーに近いサーバーでもコンテンツを配信する仕組みのことを言います。

例えば日本から `takamin55's ブログ` というブログを配信するとします。  

日本からアクセスする場合は特に気にならないでしょうが、ヨーロッパやアフリカ、アメリカなどの離れた地理の方がアクセスするとレスポンスタイムが遅くなり、イライラするかもしれません。`takamin55's blog` ならまぁ別にいいですが、企業の重要なコンテンツの配信でそれが発生すると致命的です。イライラしたユーザーはページを離れ、満足度や売り上げが低下してしまうことでしょう。

そこでユーザーの近くに、つまりヨーロッパやアフリカ、アメリカにも配信サーバーを配置するのです。そうすればユーザーは快適にコンテンツを楽しむことができます。まずこれが CDN の基本的な考え方です。

> 利点1: CDN はユーザーがコンテンツ取得にかかる待ち時間を短縮する

他にも、世界中に分散してサーバーが存在するため可用性アップにもつながっています。

> 利点2: CDN は配信サーバーを分散させるため、可用性向上にも貢献する

先ほど「世界中に分散」と書きましたが、世界中のサーバーを管理するとなると大変そうですね。CDN はどのように実現されているのでしょうか。

## CDN の仕組み・違い。Pull 型と Push 型について
CDN には Pull 型と Push 型の 2 種類が存在します。

### Pull 型 CDNの仕組みと注意点
一般的な CDN は Pull 型です。  
コンテンツがデプロイされている本体サーバー（オリジンサーバーと言う）からコンテンツを「引っ張って」きて、自身のサーバーにキャッシュします。

以下のような流れコンテンツを配信します。

1. ユーザーが配信サイトにアクセス
2. ユーザーに最も近いサーバーにルーティング
3. キャッシュがあればそれを返す。なければオリジンサーバーへリクエストしそれを返す

初回アクセス時は結局オリジンサーバーまでリクエストが必要なので、待ち時間の短縮メリットが得られません。  
またコンテンツをキャッシュするので、キャッシュ時間設定やアクセスするタイミングにより、古いコンテンツが配信されてしまう、というリスクがあります。

また、Pull 型は定期的にオリジンサーバーへのアクセスが発生します。他にもアクセスログが分散してしまう問題があります。

### Push 型 CDNの仕組みと注意点
Push 型はその名の通り明示的にプッシュ（配信）します。  
オリジンサーバーが変更されたら、世界中のサーバーにその変更を通達するのです。

Pull 型との違いとして、

- 最初からコンテンツを持つので初回アクセスも高速
- 定期的にオリジンサーバーへアクセスが発生しない。

という特徴があります。

しかし変更頻度が高いと大変ですね。Push 型は遅延が許されない場合や、変更頻度が極端に低いコンテンツ配信に向いています。
