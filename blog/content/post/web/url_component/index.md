Web の URL まわりの用語について混乱していませんか。

- URL
- スキーム
- ホスト
- オリジン
- ポート
- パス
- クエリ
- ドメイン
- FQDN

これらを整理してまとめます。

## URL を構成する要素
例えば `https://blog.takamin55.dev/page/1?lang=ja` で説明します。これは「URL（Uniform Resource Locator）」でウェブ上のリソースの位置を表します。

これはこのように分解できます。

- `https`: スキーム
- `blog.takamin55.dev`: ホスト
- `(443)`: ポート
- `/page/1`: パス
- `?lang=ja`: クエリ

そして「スキーム+ホスト+ポート」を合わせて「オリジン」と言います。`80` や `443` は省略されるのですがポートまで含みます。

オリジンの覚え方としては、オリジン = 起源なので「どこからきたのか…？」を考えるといいです。スキームとホストとポートまで含めれば同一のサービスを指しますから（パスが異なっていても同一のサービスでしょ）。

ポートも？と思うかもしれませんが、例えばポートが違っていると Admin サービスかもしれないので、ポートも含めましょう。

## ドメインと FQDN は？
`blog.takamin55.dev` を「ホスト」と呼びましたが、これは「ドメイン」では？と思ったかもしれません。

ドメインでも正解です。ただしそれは DNS の文脈の話で、URL の文脈では「ホスト」になります。

例えば、`blog.takamin55.dev` は `127.0.0.1` などの IP アドレスでも問題ないですよね。けれど IP アドレスのことをドメインとは呼ばないはずです。
なので、ホストはドメインかもしれないけれど、もっと広い意味だし、そもそも DNS の文脈の用語なので、ホストと表現する方が正確、と言えるでしょう。

FQDN は完全修飾ドメイン名（Fully Qualified Domain Name）のことで、`.` ドット、つまりルートの名前空間も含めたドメイン名を刺さす言葉です。

ドメインは階層構造になっていますが、すべてのドメインの親はルートです。

- `blog`: 3LD
  - `takamin55`: 2LD
    - `dev`: TLD
      - `.`: Root

## 終わりに
例えばコードの変数名やペアプロ時の説明などで正確な情報を伝えられるよう、正しい用語を意識しましょう。
